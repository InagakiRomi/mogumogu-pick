# 註解風格（對齊 `logging/formatter/MyCustomFormatter`）

撰寫註解時比照該檔：繁體中文、簡短、說用途或區塊在做什麼即可。

## Javadoc（`/** … */`）

- **類別與 public／private 方法**：一句話說明角色或回傳什麼（例如「自訂 log 輸出格式」「定義 log 的最終輸出格式」「根據 log 級別回傳對應的顏色」）
- 技術名詞、API 名稱可保留原文

## 行內（`//`）

- 用來標**常數區塊**（如「時間格式設定」「ANSI 顏色碼」）或**組裝／流程步驟**（如「加上時間」「加上 logger 名稱」）
- 一行一個重點；顯而易見的一行不必註解

## 避免

- 結尾加句號、長篇教學、逐行翻譯程式、與程式不同步的註解
